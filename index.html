Simulation workflow

How a simulation request moves through ROADMAP.

A public-facing request can become a reproducible compute job, a benchmark comparison, and a role-specific report without forcing users to understand the infrastructure underneath.

01

Structured request

Users select a task, model, perturbation, and target benchmark through a guided submission flow.

02

Queued on compute

The request is packaged as a job manifest and dispatched to a Compute Canada / Alliance queue.

03

Benchmark automatically

Outputs are matched to human, animal-model, and model-comparison benchmarks already stored on the platform.

04

Return useful reports

Results are turned into dashboard cards, clinician summaries, and community-readable interpretations.

ROADMAP is a knowledge-communication platform.

Its purpose is to make scientific claims more transparent, more comparable across systems, and more useful before they are carried into clinical studies, interventions, or products.
